# 脏活炼金术（Dirtywork Learning）

把非技术岗实习中重复、机械、低自主度的工作，炼成真实的业务洞察、能力沉淀、流程优化思路，以及可信的简历和面试表达。

> 核心原则：不美化无意义的消耗，也不浪费那些能够接触用户、业务流程和真实问题的重复工作。

## 项目简介

非技术岗实习经常包含大量所谓的 `dirtywork`：打电话、筛素材、复制数据、打印合同、跑腿、手工配置、竞品搜集等。

这些任务未必天然有价值。这个 Skill 的作用，是帮助你判断：

1. 这项工作背后服务的业务目标是什么；
2. 重复执行过程中能否形成有效的数据和观察；
3. 如何把机械劳动升级为结构化分析；
4. 能沉淀什么可复用成果；
5. 如何在简历和面试中真实表达，而不是强行包装；
6. 这项工作究竟值得继续学习，还是纯粹的低价值消耗。

## 适用场景

你可以输入类似内容：

- “领导让我给司机打一下午电话，这能学到什么？”
- “每天筛两个小时广告素材，感觉完全没有成长。”
- “我在手工配置一百多条优惠券，怎么从中总结东西？”
- “实习主要是打印合同和跑腿，怎么判断还有没有必要继续？”
- “帮我把这段机械工作写成真实、不夸大的简历经历。”

Skill 会从业务本质、观察指标、执行升级、成果沉淀、简历表达和价值边界六个方向进行分析。

## 输出结构

默认输出包含：

```text
这项 dirtywork 的表层任务
它背后的业务本质
可以重点观察什么
怎么把它做得更聪明
最终沉淀
简历 / 面试表达
边界判断
```

## 示例

### 输入

> 我在实习中需要给司机打一下午电话，询问他们为什么近期没有接单。

### 输出思路

- **业务本质**：司机供给侧用户研究与流失原因分析；
- **重点观察**：不接单原因、产品功能障碍、补贴敏感度、接单场景、重复投诉；
- **执行升级**：设计半结构化访谈提纲，为反馈建立标签并记录频次；
- **成果沉淀**：形成司机流失原因分类表和典型原话库；
- **简历表达**：参与司机侧用户访谈，整理高频不接单原因并输出问题分类，为运营策略和产品优化提供反馈；
- **边界判断**：只有在反馈被整理、讨论或应用时，才具有较高学习价值。纯打电话且无人反馈结果，成长性有限。

## 价值判断框架

Skill 会从四个维度对任务进行判断，每项 0–2 分：

| 维度 | 判断问题 |
|---|---|
| 业务可见度 | 能否接触用户、收入、风险、系统或决策？ |
| 模式密度 | 重复执行能否暴露有意义的规律？ |
| 反馈机会 | 是否能与有经验的人讨论发现？ |
| 改进空间 | 流程、分析或输出能否被优化？ |

- **6–8 分**：学习潜力较高，值得主动结构化和沉淀；
- **3–5 分**：价值混合，建议产出一项成果后争取更丰富任务；
- **0–2 分**：主要是低价值劳动，不应过度浪漫化。

## 使用方式

### 在 ChatGPT 中安装

1. 下载本仓库；
2. 将 `dirtywork-learning` 目录打包为 ZIP；
3. 在 ChatGPT 的 Skills 页面上传；
4. 描述一项实习任务并让 ChatGPT 分析。

仓库中的核心文件：

```text
dirtywork-learning/
├── SKILL.md
├── agents/
│   └── openai.yaml
└── references/
    └── framework.md
```

## 设计原则

- 不把简单执行夸大成“战略负责”；
- 不虚构业务结果和数据；
- 区分参与、观察、提出建议、设计方案和独立负责；
- 把重复劳动视为潜在数据来源，但不默认它一定有学习价值；
- 鼓励形成分类表、流程图、分析备忘录、需求草案等实际成果；
- 当实习长期缺少指导、反馈和任务变化时，明确提醒重新评估实习价值。
