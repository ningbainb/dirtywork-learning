# Dirtywork Analysis Framework

## Core Questions

1. What business outcome is this task ultimately serving?
2. What information becomes visible only after doing it many times?
3. Which patterns can be categorized or counted?
4. Where do errors, delays, complaints, or rework occur?
5. Which teams, systems, or incentives create the current process?
6. What can an intern improve without pretending to own the whole strategy?
7. What artifact would make the learning reusable for others?

## Value Test

Classify the task using four dimensions, each from 0 to 2:

- **business visibility**: does it reveal users, revenue, risk, systems, or decisions?
- **pattern density**: does repetition expose meaningful recurring signals?
- **feedback access**: can the intern discuss findings with someone knowledgeable?
- **improvement room**: can the process, analysis, or output be made better?

Interpretation:
- 6-8: high learning potential; actively structure and document it.
- 3-5: mixed value; extract one artifact, then seek broader work.
- 0-2: mostly low-value labor; do not over-romanticize it.

## Resume Language Ladder

Use the strongest wording supported by evidence:

- executed / assisted / maintained
- analyzed / summarized / identified
- proposed / designed / built
- coordinated / led / owned

Never jump directly from "copied data" to "led growth strategy." A truthful progression might be:

"Maintained 100+ coupon configurations, documented recurring setup errors, mapped cross-platform dependencies, and proposed requirements for a batch-configuration workflow."
