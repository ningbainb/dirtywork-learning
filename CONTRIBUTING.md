# 贡献指南

感谢你愿意改进 Dirtywork Learning。

## 可以贡献什么

- 新增典型实习场景及分析方法；
- 优化 `SKILL.md` 中的工作流；
- 补充行业或岗位案例；
- 改进价值判断标准；
- 修正文案、结构或表达问题；
- 增加更真实且不夸大的简历示例。

## 贡献原则

1. **保持真实**：不得虚构数据、结果或职责范围。
2. **拒绝过度包装**：不要把执行工作直接描述为战略主导。
3. **避免美化剥削**：不得暗示实习生必须忍受长期无指导、无成长、无关岗位或不安全的工作。
4. **强调可操作性**：建议应当是普通实习生能够执行的小步改进。
5. **保护隐私**：案例中不要出现真实用户的敏感信息或公司的未公开信息。

## 提交流程

1. Fork 本仓库；
2. 创建分支，例如 `feat/new-scenario`；
3. 修改并检查 Markdown 格式；
4. 提交清晰的 Commit；
5. 发起 Pull Request，并说明修改内容和原因。

## 案例建议格式

```markdown
### 场景名称

- 表层任务：
- 业务本质：
- 可观察信号：
- 执行升级：
- 可沉淀成果：
- 真实表达：
- 边界判断：
```
